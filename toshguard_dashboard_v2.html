How it works

  • We read LiquidityLocked and LiquidityUnlocked events to discover (token,user) pairs.
  • Then we call the contract getter for each pair to read the current locked amount and unlockTime.
  • Only pairs with a non-zero amount are shown.

Tip: set From block to your contract's deploy block to scan faster.
